Alkebulan WebSocket Routes

Description:
The Alkebulan document retrieval and processing system provides real-time chat functionality via WebSocket connections. This allows clients to interact with the system for retrieving, processing, and receiving insights from documents using large language models (LLMs). The WebSocket chat routes offer bi-directional communication, enabling seamless queries and responses as well as processing updates in real-time.

Routes Overview:

Alkebulan's WebSocket chat routes facilitate the following:

  • Document Retrieval: Query documents from the system's extensive database using LLMs for natural language processing.
  • Real-Time Responses: Receive real-time responses and processing feedback on document-based queries.
  • Interactive Chat: Engage in an interactive chat experience where each query is processed in real-time by the underlying LLM models.

WebSocket Endpoints:

1. Chat Route for Querying Winter Sports Data

  • Endpoint: ws://{baseUrl}/chat/similarity-search
  • Description: This WebSocket route allows users to submit queries related to winter sports data. The data is stored in a Qdrant vector database, and upon retrieval, the WebSocket connection returns relevant information based on the query. This route is ideal for retrieving document embeddings and insights related to winter sports.
  • Usage:
    • Send queries related to winter sports (e.g., "List the most successful Olympic winter sports athletes").
    • Receive real-time results based on vector similarity search.
  • Payload Format:
    • Request: Plain text containing the query (e.g., "Top winter sports events of 2022").
    • Response: Plain text response corresponding to the query (e.g., "The top events of 2022 include ...").

2. Chat Route for Querying Financial Data

  • Endpoint: ws://{baseUrl}/chat/open-search
  • Description: This WebSocket route allows users to query financial data stored in an OpenSearch database deployed on AWS. Clients can send financial-related queries and receive real-time responses based on indexed financial documents and data from the OpenSearch database.
  • Usage:
    • Send financial data queries (e.g., "Get stock prices for 2023" or "Fetch quarterly reports for XYZ Corp").
    • Receive real-time results by querying the OpenSearch index on AWS.
  • Payload Format:
    • Request: Plain text containing the query (e.g., "Quarterly financial report for XYZ Corp").
    • Response: Plain text response containing financial data or documents corresponding to the query (e.g., "XYZ Corp Q1 2023 report shows ...").
